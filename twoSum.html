01
Two Sum
Pattern: HashMap Complement Lookup DS: Array + HashMap 🔥 Critical — Every Interview
// problem statement

Given an array of integers nums and integer target, return the indices of the two numbers that add up to target. Each input has exactly one solution. You may not use the same element twice.

// examples
Input
nums = [2, 7, 11, 15]
target = 9
Output
[0, 1] // 2+7=9
Input
nums = [3, 2, 4]
target = 6
Output
[1, 2] // 2+4=6
// approach
❌ Brute Force — Don't do this
Two nested loops check every pair. Mentions this, then immediately say you can do better.
Time: O(n²)Space: O(1)
✅ HashMap — O(n)
For each number, look up its complement (target − current) in a map. Store what you've seen.
Time: O(n)Space: O(n)
// optimal solution
javascript 
// ✅ HashMap approach — O(n) time, O(n) space
function twoSum(nums, target) {
  const map = {};  // number → index

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];

    // If complement already seen → answer found!
    if (map[complement] !== undefined) {
      return [map[complement], i];
    }

    // Store current number with its index
    map[nums[i]] = i;
  }
  return [];
}

// Tests
console.log(twoSum([2, 7, 11, 15], 9));  // [0, 1]
console.log(twoSum([3, 2, 4], 6));        // [1, 2]
console.log(twoSum([3, 3], 6));            // [0, 1]
// step-by-step dry run — [2,7,11,15], target=9

179−7 = 2{2:0}✅ 2 IS in map → return [0, 1]
⚠️
Common Trap
Use map[complement] !== undefined — not just !map[complement]. If the answer index is 0, it's falsy and would be missed!
💡
Interview Tip
"Store what you've seen, look up what you need" — this is the HashMap pattern. Mentioning it by name impresses interviewers.
02
Contains Duplicate
Pattern: HashSet O(1) Lookup DS: Array + HashSet 🔥 Critical
// problem statement

Return true if any value appears at least twice in nums, or false if every element is distinct.

// examples
Input
nums = [1, 2, 3, 1]
Output
true // 1 appears twice
Input
nums = [1, 2, 3, 4]
Output
false // all distinct
// solution
javascript 
// ✅ HashSet — returns early on first duplicate
function containsDuplicate(nums) {
  const seen = new Set();

  for (const num of nums) {
    if (seen.has(num)) return true;
    seen.add(num);
  }
  return false;
}

// One-liner (less efficient — no early exit)
const hasDup = (nums) => nums.length !== new Set(nums).size;

// Variant: Duplicate within K distance
function containsNearbyDuplicate(nums, k) {
  const map = new Map(); // value → last index
  for (let i = 0; i < nums.length; i++) {
    if (map.has(nums[i]) && i - map.get(nums[i]) <= k)
      return true;
    map.set(nums[i], i);
  }
  return false;
}
Time
O(n)
Space
O(n)
The loop version returns early on first duplicate — faster in practice than the one-liner which builds the full Set first.
03
First Unique Character in String
Pattern: Two-Pass Frequency Counter DS: String + HashMap ⚡ High
// problem statement

Given string s, return the index of the first non-repeating character. Return -1 if none exists.

// examples
Input
"leetcode"
Output
0 // 'l' appears once
Input
"loveleetcode"
Output
2 // 'v' is first unique
// solution — two pass
javascript 
function firstUniqChar(s) {
  const freq = {};

  // Pass 1: count each character
  for (const char of s) {
    freq[char] = (freq[char] || 0) + 1;
  }

  // Pass 2: find first character with count = 1
  for (let i = 0; i < s.length; i++) {
    if (freq[s[i]] === 1) return i;
  }

  return -1;
}

console.log(firstUniqChar('leetcode'));      // 0
console.log(firstUniqChar('loveleetcode'));  // 2
console.log(firstUniqChar('aabb'));          // -1
Time
O(n)
Space
O(1)
O(1) space because there are only 26 lowercase letters. The frequency map caps at 26 entries — constant regardless of string length.
04
Group Anagrams
Pattern: Sort → HashMap Key DS: Array + HashMap 🔥 Critical — Very Important
// problem statement

Given an array of strings, group all anagrams together. An anagram uses all the original letters exactly once in a different order.

// examples
Input
["eat","tea","tan",
"ate","nat","bat"]
Output
[["eat","tea","ate"],
["tan","nat"],["bat"]]
// key insight
🧠
The Insight
Anagrams share the same sorted character string. "eat", "tea", "ate" all sort to "aet". Use that as the HashMap key.
// solution
javascript 
// ✅ Sort-based key — clean and readable
function groupAnagrams(strs) {
  const map = {};

  for (const word of strs) {
    // "eat" → split → ['e','a','t'] → sort → ['a','e','t'] → "aet"
    const key = word.split('').sort().join('');
    if (!map[key]) map[key] = [];
    map[key].push(word);
  }

  return Object.values(map);
}

// ✅ Frequency array key — O(k) per word instead of O(k log k)
function groupAnagramsFast(strs) {
  const map = new Map();
  for (const word of strs) {
    const freq = new Array(26).fill(0);
    for (const c of word) freq[c.charCodeAt(0) - 97]++;
    const key = freq.join('#'); // "1#0#0...1#0#0..."
    if (!map.has(key)) map.set(key, []);
    map.get(key).push(word);
  }
  return [...map.values()];
}

console.log(groupAnagrams(['eat','tea','tan','ate','nat','bat']));
// [['eat','tea','ate'], ['tan','nat'], ['bat']]
Sort Key
O(n·k log k)
Freq Key
O(n·k)
Mentioning the frequency array key as an optimization earns extra credit in senior interviews. Both solutions are accepted.
05
Longest Substring Without Repeating Characters
Pattern: Sliding Window + HashMap DS: String 🔥 Critical — Frontend Favourite
// problem statement

Find the length of the longest substring without repeating characters.

// examples
Input
"abcabcbb"
Output
3 // "abc"
Input
"bbbbb"
Output
1 // "b"
Input
"pwwkew"
Output
3 // "wke"
// solution — sliding window
javascript 
function lengthOfLongestSubstring(s) {
  let left = 0;
  let max = 0;
  const map = {};  // char → last seen index

  for (let right = 0; right < s.length; right++) {
    const char = s[right];

    // char is inside current window → shrink from left
    if (map[char] >= left) {
      left = map[char] + 1;
    }

    map[char] = right;  // update last seen
    max = Math.max(max, right - left + 1);
  }

  return max;
}

console.log(lengthOfLongestSubstring('abcabcbb'));  // 3
console.log(lengthOfLongestSubstring('pwwkew'));    // 3

Time
O(n)
Space
O(min(m,n))
m = charset size. For lowercase English: O(26) = O(1). For Unicode: O(n).
⚠️
Critical Trap
Must check map[char] >= left — NOT just map[char]. Without >= left, you might jump left backwards to before the current window!
06
Subarray Sum Equals K
Pattern: Prefix Sum + HashMap DS: Array + HashMap ⚡ High — Advanced Level
// problem statement

Return the total number of subarrays whose sum equals k. Works with negative numbers!

// examples
Input
nums = [1, 1, 1], k = 2
Output
2 // [1,1]×2
Input
nums = [1, 2, 3], k = 3
Output
2 // [1,2] and [3]
// key insight
🧠
The Insight
Sum of nums[i..j] = prefix[j+1] − prefix[i]. We want this = k, so we need prefix[j+1] − k to have appeared earlier. Track prefix sum frequencies in a HashMap.
// solution
javascript 
function subarraySum(nums, k) {
  const prefixCount = { 0: 1 };  // ← CRITICAL: handles subarrays starting at index 0
  let runningSum = 0;
  let count = 0;

  for (const num of nums) {
    runningSum += num;

    // Have we seen (runningSum - k) before?
    // If yes → a subarray ending here has sum = k
    const needed = runningSum - k;
    if (prefixCount[needed]) {
      count += prefixCount[needed];
    }

    prefixCount[runningSum] = (prefixCount[runningSum] || 0) + 1;
  }

  return count;
}

console.log(subarraySum([1,1,1], 2));   // 2
console.log(subarraySum([1,2,3], 3));   // 2
console.log(subarraySum([1,-1,1], 0));  // 3  ← handles negatives!

Time
O(n)
Space
O(n)
Handles negatives! Sliding window only works for positive arrays. This prefix sum approach works universally.
⚠️
Don't Forget
Initialize prefixCount = { 0: 1 }. Without this, subarrays that start from index 0 (where the prefix sum itself equals k) won't be counted.
