The Two Duplicate Checks Explained

❌ Without Check — Duplicate Triplets

Without skipping duplicates, the same triplet gets recorded multiple times. For [-1,-1,0,1,2] both -1s at index 0 and 1 produce [-1,0,1] — a duplicate.

// ❌ Missing outer skip for (let i = 0; i < n-2; i++) { // if (i>0 && nums[i]===nums[i-1]) continue ... } // ❌ Missing inner skip if (sum === 0) { result.push([...]); // while(nums[left]===nums[left+1]) left++ // while(nums[right]===nums[right-1]) right-- left++; right--; }
✅ With Check — Unique Triplets Only

Skipping duplicate i values avoids reprocessing the same fixed anchor. After finding a triplet, skipping equal left/right values prevents recording the same pair again.

// ✅ Outer skip for i if (i > 0 && nums[i] === nums[i-1]) continue; // ✅ Inner skip after finding triplet if (sum === 0) { result.push([...]); while(nums[left]===nums[left+1]) left++; while(nums[right]===nums[right-1]) right--; left++; right--; }
